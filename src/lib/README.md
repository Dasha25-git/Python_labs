# Лабораторная работа №1

В этом фрагменте создаём класс Book с атрибутом класса currency и конструктором __init__, который принимает основные характеристики книги. Перед сохранением данных выполняется их проверка: строки не должны быть пустыми, числа — целыми и в допустимом диапазоне, цена — неотрицательная, а доступность — логического типа. После успешной валидации значения сохраняются в закрытые поля объекта, при этом строки очищаются от лишних пробелов, а цена приводится к типу float. Таким образом обеспечивается корректность данных и реализуется инкапсуляция.

![Скриншот1](../../images/lab01/image1.png)

В данном фрагменте реализованы свойства класса с использованием декоратора @property. Каждый метод (title, author, year, pages, available, price) возвращает соответствующее закрытое поле объекта (_title, _author и т.д.). Благодаря @property эти методы можно использовать как обычные атрибуты (например, book.title. Все свойства предназначены только для чтения.

![Скриншот2](./src/images/lab01/image2.png)

В данном фрагменте реализован setter для свойства price. Setter позволяет изменять цену книги, при этом новое значение предварительно проходит проверку через метод валидации и приводится к типу float. Метод __str__ отвечает за красивый вывод объекта при использовании print(): он формирует строку с основной информацией о книге, включая статус доступности. Метод __repr__ возвращает техническое представление объекта. Метод __eq__ определяет логику сравнения двух книг — они считаются равными, если совпадают название, автор (без учёта регистра) и год издания.

![Скриншот3](./src/images/lab01/image3.png)

В данном фрагменте реализованы бизнес-методы класса. Метод apply_discount применяет к цене книги процентную скидку: сначала проверяется, что процент является числом и находится в диапазоне от 0 до 100, затем рассчитывается новая цена по формуле, после чего она сохраняется через setter и возвращается. Метод mark_borrowed помечает книгу как выданную, предварительно проверяя, что она ещё доступна; если книга уже выдана, возникает ошибка. Метод mark_returned выполняет обратное действие — делает книгу доступной, но только если она действительно была выдана.

![Скриншот4](./src/images/lab01/image4.png)

В данном фрагменте реализованы статические методы валидации, которые проверяют корректность входных данных. Метод _validate_nonempty_str проверяет, что значение является непустой строкой, _validate_bool — что значение логического типа, _validate_int_range — что передано целое число в заданном диапазоне, а _validate_price — что цена является числом и находится в допустимых пределах.
